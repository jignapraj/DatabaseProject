# Insurance Company Database Project
# Project Overview
_This project focuses on designing and implementing a relational database system for an insurance company. Our goal was to create a structured and efficient database that mimics real-world insurance operations, including policy management, invoicing, and data integrity enforcement._

# Key Features & Deliverables
**Entity-Relationship Diagram (ERD):**

Designed a comprehensive ERD to represent the relationships between policies, customers, claims, and invoices. 

**Data Analysis & Realistic Values:**

Conducted in-depth research to add real-world values to insurance policies, ensuring practical implementation.

**Database Normalization:**

Applied normalization techniques up to the third normal form (3NF) to optimize data consistency and minimize redundancy.

**SQL Implementation:**

Created tables using CREATE TABLE statements.

Inserted sample data using INSERT statements.

Applied ALTER TABLE statements to modify schema when needed.

Added constraints such as PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, and CHECK for data integrity.

**Backend Development:**

Developed queries and procedures to interact with the database efficiently.

**Testing & Validation:**

Executed SQL queries to verify the correctness of database transactions and ensure reliability.

Ensuring proper relationships and applying normalization required multiple iterations and refinements.
